プロローグ：ダブルキーの金庫

情シス部での研修が始まって1週間。
アオキ（25）は、重要書類が保管されているというオフィス奥のスチール金庫の前にいた。
金庫の扉には、不自然に2つの鍵穴が並んでいる。
【ep03_01.png】2つの鍵穴を持つ情シス部の重要金庫
アオキ
「あれ？ この金庫、鍵穴が2つありますね。私の持っている『管理者の鍵』だけじゃ開かないのかな……」
ホシノ
「ええ、その金庫はセキュリティ確保のため、アオキさんの持つ『管理者の鍵』と、私が持つ『承認者の鍵』の**両方を差し込んで回さないと開かない**仕様になっております」
ミズダイ
「うむ！ まさにこれこそが**AND（論理積）**の精神！ 片方の鍵がある（1）だけではダメ！ 両方の鍵が揃って（1かつ1）初めて扉が開く（1）のだ！ 美しいセキュリティフォーメーションだな！」
アオキ
「なるほど。論理演算のANDって、すべての条件が『1（真）』の時だけ、結果が『1（真）』になる演算のことでしたよね。」
ホシノ
「その通りです。コンピュータでは以下のように処理されます。
  • 『アオキの鍵あり（1）』 AND 『ホシノの鍵なし（0）』 ＝ 『開かない（0）』
  • 『アオキの鍵あり（1）』 AND 『ホシノの鍵あり（1）』 ＝ 『開く（1）』
このように、すべての入力が 1 のときのみ 1 を出力するのがAND演算です。」

お支払いは現金かカードか

お昼休み。情シス部の3人は社内売店にやってきた。
レジの前に立ったミズダイ課長が、突然ポケットを叩いて絶叫した。
ミズダイ
「しまったァァ！ 財布をデスクに忘れた！ しかし、私のスマホには電子マネーカードが登録されているッ！ ふぅ、危うくランチ難民になるところだったぞ！」
アオキ
「心臓に悪いので大声出すのやめてください。……でも、買い物は『現金』か『カード（電子マネー）』のどちらか一方でもあれば決済できますよね。」
ホシノ
「そうですね。これが**OR（論理和）**の仕組みです。OR演算は、複数の条件のうち**少なくとも1つが『1（真）』**であれば、結果が『1（真）』になります。」
【ep03_02.png】現金かカードのどちらかがあれば決済できる（ORの例え）
ホシノ
「お買い物で例えると以下のようになります。
  • 『現金なし（0）』 OR 『カードあり（1）』 ＝ 『買い物できる（1）』
  • 『現金あり（1）』 OR 『カードあり（1）』 ＝ 『買い物できる（1）』
  • 『現金なし（0）』 OR 『カードなし（0）』 ＝ 『買い物できない（0）』
どちらか片方、あるいは両方の入力が 1 であれば 1 を出力するのがOR演算です。」

暗くなったらON

夕方、オフィスに戻ると、外が薄暗くなってきた。
すると、窓際のセンサーライトが自動的にフワッと点灯した。
アオキ
「あ、電気がついた。情シスのセンサーライトって自動なんですね。」
ホシノ
「はい、明るさセンサーが外の光を検知しているのです。明るさが『明るくない（0）』状態になると、ライトが『点灯（1）』する仕組みになっています」
ミズダイ
「これぞ**NOT（否定）**の極み！ 入力をそっくりそのまま反転させるのだ！ 『明るい（1）』をNOTすると『消灯（0）』！ 『明るくない（0）』をNOTすると『点灯（1）』！」
アオキ
「一番シンプルですね。入力された値をそのまま逆にするだけ、と。」
【ep03_03.png】入力を反転させるNOT演算の仕組み

階段の電灯と3路スイッチ

アオキ
「そういえば、うちのオフィスの階段の電気なんですけど……。1階でスイッチを押して電気を点けて、2階に上がって別のスイッチを押したら電気が消えますよね。あれってどういう回路なんですか？」
ホシノ
「おや、良い疑問ですね。あれは『3路スイッチ』と呼ばれる配線で、コンピュータの論理回路では**XOR（排他的論理和）**と同じ働きをしています。」
【ep03_04.png】上と下のスイッチが『異なる状態』のときだけ点灯する階段のライト
ホシノ
「XOR演算は、**2つの入力が『異なるとき』だけ結果が『1（真）』**になり、**『同じとき』は結果が『0（偽）』**になります。
階段のスイッチで言うと、以下のようになります。
  • 『1階スイッチ：上向き（1）』 XOR 『2階スイッチ：下向き（0）』 ＝ 『点灯（1）』
  • 『1階スイッチ：上向き（1）』 XOR 『2階スイッチ：上向き（1）』 ＝ 『消灯（0）』
このように、どちらか片方だけが ON の状態のときだけ電気が通る仕組みになっているのです」
ミズダイ
「『排他的（Exclusive）』とは相手を排除する、すなわち『重複を許さない』ということ！ 2つのスイッチが揃って同じ方向（1と1、あるいは0と0）を向いている贅沢者めらは、XORの鉄槌によって0（消灯）へと叩き落とされるのだ！」
アオキ
「相変わらず例えが過激ですけど、イメージはすごく分かりやすいです。同じ向きだとOFF、違う向きだとONになるのがXORですね。」

午前問題（科目A）にチャレンジ！

ホシノさんがホワイトボードに、論理演算に関する試験問題を書き出した！
【問題1】
2つの入力A、Bに対する出力Xの真理値表が以下のようになるとき、この論理演算を示すものはどれか。

ABX
000
011
101
110

ア：AND
イ：OR
ウ：XOR
エ：NOT
アオキ
「表を見ると……
・AとBが『0と0』のときは『0』
・AとBが『0と1』、または『1と0』のように『値が異なるとき』は『1』
・AとBが『1と1』のときは『0』
値が異なるときだけ1が出力されているから、これはさっき習った階段のスイッチと同じ！ 正解は **『ウのXOR』** です！」
ホシノ
「素晴らしい！ 大正解です。この表を**真理値表（しんりちひょう）**と呼び、試験では表を読み取る問題が非常によく出題されます」
【問題2】
2つのビット列 10101100 と 01101010 に対し、ビットごとの排他的論理和（XOR）演算を行った結果はどれか。

ア：00101000
イ：11101110
ウ：11000110
エ：00010001
ミズダイ
「ほう！ 今度はビット列同士の演算だな！ アオキ、XORの奥義をビット列に叩き込んでみせよ！」
アオキ
「ビットごとの演算ですね。上と下のビットを1桁ずつ見比べて、XOR（値が異なれば1、同じなら0）を計算していきます。
・1桁目（左端）：1 と 0 → 異なるから 1
・2桁目：0 と 1 → 異なるから 1
・3桁目：1 と 1 → 同じだから 0
・4桁目：0 と 0 → 同じだから 0
・5桁目：1 と 1 → 同じだから 0
・6桁目：1 と 0 → 異なるから 1
・7桁目：0 と 1 → 異なるから 1
・8桁目（右端）：0 と 0 → 同じだから 0
これらを並べると `11000110` になるので、正解は **『ウ』** です！」
ホシノ
「お見事、大正解です！ ちなみに、選択肢『ア』はAND（論理積）の演算結果、『イ』はOR（論理和）の演算結果になっております。こちらもあわせて確認しておくと理解が深まりますよ」
ミズダイ
「ガハハ！ 論理の迷宮をいともたやすく突破するとは！ アオキ、お前の脳内トランジスタ（回路）も良い具合に繋がってきたようだな！ よし、今日の褒美に、情シス特製のプロテインドリンクをプレゼントしよう！」
アオキ
「プロテインは遠慮しますって言ってますよね……。でも、真理値表の読み方さえわかれば、パズルみたいにサクサク解けて楽しいです！」
論理演算の基本であるAND、OR、NOT、XORをマスターしたアオキ。
0と1の組み合わせから生まれるコンピュータの知恵に触れ、情シス部での歩みはさらに確かなものとなっていく！
